## 1. 设计原则

### 1.1 MVP简化原则

**核心思想**：MVP阶段表结构越简单越好，能存PVC的就不存数据库。

| 原则 | 说明 |
|-----|------|
| **最小化表数量** | 仅保留7张核心表，满足基本游戏需求 |
| **复用OpenClaw能力** | 用户记忆、对话历史由OpenClaw管理，存PVC |
| **避免过度设计** | 不预留未来功能的表结构 |
| **快速迭代** | 表结构简单，便于修改和扩展 |
| **子项目不创建独立表** | 所有游戏相关数据由主项目管理 |
| **每用户独立沙箱** | 每个用户拥有独立的OpenClaw Pod，数据隔离 |
| **强隔离原则** | 每用户独立容器，严禁多用户共享同一个OpenClaw进程 |

### 1.2 数据存储分工

```
┌─────────────────────────────────────────────────────────────────┐
│                     数据存储分工（7张核心表）                    │
├─────────────────────────────────────────────────────────────────┤

│  ├── users（用户基本信息）                                       │
│  ├── ai_partners（AI状态：记忆点数、性格、成长阶段）              │
│  ├── story_progress（剧情进度）                                  │
│  ├── memory_points_log（记忆点数变更记录）                       │
│  ├── daily_signin（每日签到记录）                                │
│  ├── consensus_votes（共识投票记录）                             │
│  └── deep_dialogue（深度对话记录）                              │
│                                                                 │
│  PVC（华为云EVS）- OpenClaw管理的数据                            │
│  └── /home/node/.openclaw/workspace/                            │
│      ├── conversations/（对话历史 - JSONL格式）                  │
│      ├── memory/（长期记忆文件 - Markdown格式）                  │
```

### 1.3 为什么不需要更多表？

| 原表 | 为什么移除 | 数据存储位置 |
|-----|-----------|-------------|
| `interactions` | OpenClaw已管理对话历史 | PVC (`conversations/`) |
| `memories` | OpenClaw已管理用户记忆 | PVC (`memory/`) |
| `stories` | 剧情骨架可硬编码或存文件 | 代码/配置文件 |
| `daily_signin` | 独立为daily_signin表 | `daily_signin` |
| `weekly_evaluations` | 简化为内存点数日志 | `memory_points_log` |
| `milestone_rewards` | 可从memory_points计算 | 无需存储 |
| `social_shares` | 简化为memory_points_log | `memory_points_log` |
| `achievements` | 后续版本 | - |
| `onboarding_states` | 简化到users表 | `users.onboarding_step` |
| `ai_events` | 过度设计 | - |

## 2. 数据存储策略

### 2.1 Supabase存储（需要查询和统计的数据）

| 数据类型 | 存储原因 | 表名 |
|---------|---------|------|
| 用户基本信息 | 需要查询、认证 | `users` |
| AI伙伴状态 | 需要频繁查询和更新 | `ai_partners` |
| 记忆点数 | 需要统计、排行榜 | `ai_partners.memory_points` |
| 剧情进度 | 需要查询当前进度 | `story_progress` |
| 点数变更记录 | 需要审计、统计 | `memory_points_log` |
| 每日签到记录 | 需要统计连续签到天数、奖励 | `daily_signin` |
| 共识投票记录 | 需要记录用户投票选择 | `consensus_votes` |
| 深度对话记录 | 需要记录深度对话完成状态、奖励 | `deep_dialogue` |

### 2.2 PVC存储（OpenClaw管理的数据）

| 数据类型 | 存储位置 | 管理方 |
|---------|---------|--------|
| 对话历史 | `workspace/conversations/*.jsonl` | OpenClaw自动管理 |
| 短期记忆 | `workspace/conversations/*.jsonl` | OpenClaw自动管理 |
| 长期记忆 | `workspace/memory/*.md` | OpenClaw Skills管理 |
| 用户偏好 | `workspace/memory/PREFERENCES.md` | OpenClaw Skills管理 |
| AI配置 | `workspace/config/personality.yml` | 主项目写入 |

### 2.3 数据同步机制

主项目通过API将用户上下文传递给OpenClaw：

```typescript
interface UserContext {
  userId: string;
  memoryPoints: number;           // 从Supabase读取
  personalityType: string;        // 从Supabase读取
  growthStage: string;            // 从Supabase读取
  currentStoryChapter: number;    // 从Supabase读取
}
```

OpenClaw通过Skills访问PVC中的记忆文件，无需数据库查询。

```sql
-- ============================================
-- users表（用户基本信息）

CREATE TABLE IF NOT EXISTS public.users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- 用户标识
  telegram_user_id BIGINT UNIQUE NOT NULL,
  telegram_username VARCHAR(100),
  
  -- 游戏进度（简化）
  onboarding_step INTEGER DEFAULT 0,  -- 0=未开始, 1=阶段1, 2=阶段2, 3=阶段3, 4=完成
  onboarding_completed BOOLEAN DEFAULT FALSE,
  
  -- 登录信息
  last_login_at TIMESTAMPTZ DEFAULT NOW(),
  consecutive_login_days INTEGER DEFAULT 0,
  
  -- OpenClaw Pod信息
  openclaw_pod_name TEXT,
  openclaw_pod_status VARCHAR(20) DEFAULT 'pending',  -- pending, running, stopped
  
  -- 时间戳
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_users_telegram_user_id ON public.users(telegram_user_id);
CREATE INDEX IF NOT EXISTS idx_users_last_login ON public.users(last_login_at);

-- RLS策略
ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own profile" ON public.users
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Service role full access" ON public.users
  FOR ALL USING (auth.role() = 'service_role');

-- 注释
COMMENT ON TABLE public.users IS '用户基本信息表（简化版）';
COMMENT ON COLUMN public.users.onboarding_step IS '新手引导步骤：0=未开始, 1=情感建立, 2=机制引入, 3=目标建立, 4=完成';
COMMENT ON COLUMN public.users.openclaw_pod_name IS '用户OpenClaw Pod名称';
COMMENT ON COLUMN public.users.openclaw_pod_status IS 'Pod状态：pending, running, stopped';
```

### 3.2 ai_partners表（AI伙伴状态）

```sql
-- ============================================
-- ai_partners表（AI伙伴状态）
-- ============================================
CREATE TABLE IF NOT EXISTS public.ai_partners (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  
  -- 基本信息
  name VARCHAR(100) DEFAULT 'AI伙伴',
  personality VARCHAR(20) DEFAULT 'warm',  -- warm, humorous, rational, lively
  CHECK (personality IN ('warm', 'humorous', 'rational', 'lively')),
  
  -- 核心状态（合并多个字段）
  memory_points INTEGER DEFAULT 0 CHECK (memory_points >= 0),  -- 当前记忆点数，非负约束
  growth_stage VARCHAR(20) DEFAULT '懵懂期',  -- 懵懂期/成长期/成熟期/觉醒期
  CHECK (growth_stage IN ('懵懂期', '成长期', '成熟期', '觉醒期')),
  current_title VARCHAR(50) DEFAULT '初识',
  
  -- 性格倾向（剧情选择累积）
  personality_scores JSONB DEFAULT '{"emotional": 0, "rational": 0, "adventurous": 0}',
  CHECK (
    jsonb_typeof(personality_scores) = 'object' AND
    jsonb_exists(personality_scores, 'emotional') AND
    jsonb_exists(personality_scores, 'rational') AND
    jsonb_exists(personality_scores, 'adventurous')
  ),
  
  -- AI状态
  status VARCHAR(20) DEFAULT 'normal',  -- normal, happy, sad, worried, lonely, dormant
  CHECK (status IN ('normal', 'happy', 'sad', 'worried', 'lonely', 'dormant')),
  
  -- 能力解锁（基于记忆点数里程碑）
  abilities JSONB DEFAULT '{
    "basic_chat": true,
    "emotion_expression": false,
    "task_system": false,
    "exclusive_memory": false,
    "deep_conversation": false,
    "self_awareness": false
  }',
  CHECK (
    jsonb_typeof(abilities) = 'object' AND
    jsonb_exists(abilities, 'basic_chat') AND
    jsonb_exists(abilities, 'emotion_expression') AND
    jsonb_exists(abilities, 'task_system') AND
    jsonb_exists(abilities, 'exclusive_memory') AND
    jsonb_exists(abilities, 'deep_conversation') AND
    jsonb_exists(abilities, 'self_awareness')
  ),
  
  -- 休眠相关
  dormant_since TIMESTAMPTZ,
  CHECK (
    (status = 'dormant' AND dormant_since IS NOT NULL) OR
    (status != 'dormant' AND dormant_since IS NULL)
  ),
  
  -- 时间戳
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  last_interaction_at TIMESTAMPTZ DEFAULT NOW(),
  CHECK (last_interaction_at >= created_at),
  
  UNIQUE(user_id)
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_ai_partners_user_id ON public.ai_partners(user_id);
CREATE INDEX IF NOT EXISTS idx_ai_partners_status ON public.ai_partners(status);
CREATE INDEX IF NOT EXISTS idx_ai_partners_memory_points ON public.ai_partners(memory_points DESC);
CREATE INDEX IF NOT EXISTS idx_ai_partners_growth_stage ON public.ai_partners(growth_stage);
CREATE INDEX IF NOT EXISTS idx_ai_partners_personality ON public.ai_partners(personality);
-- JSONB字段的GIN索引（用于JSONB查询优化）
CREATE INDEX IF NOT EXISTS idx_ai_partners_abilities_gin ON public.ai_partners USING gin(abilities);
CREATE INDEX IF NOT EXISTS idx_ai_partners_personality_scores_gin ON public.ai_partners USING gin(personality_scores);
-- 复合索引（优化常见查询）
CREATE INDEX IF NOT EXISTS idx_ai_partners_status_points ON public.ai_partners(status, memory_points DESC);
CREATE INDEX IF NOT EXISTS idx_ai_partners_user_stage ON public.ai_partners(user_id, growth_stage);

-- RLS策略
ALTER TABLE public.ai_partners ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view own AI partner" ON public.ai_partners
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Service role full access" ON public.ai_partners
  FOR ALL USING (auth.role() = 'service_role');

-- 注释
COMMENT ON TABLE public.ai_partners IS 'AI伙伴状态表（核心状态合并版）';
COMMENT ON COLUMN public.ai_partners.memory_points IS '当前记忆点数（情感连接深度指标）';
COMMENT ON COLUMN public.ai_partners.growth_stage IS '成长阶段：懵懂期(0-50)/成长期(51-200)/成熟期(201-500)/觉醒期(500+)';
COMMENT ON COLUMN public.ai_partners.abilities IS '已解锁能力，基于记忆点数里程碑自动解锁';
COMMENT ON COLUMN public.ai_partners.dormant_since IS '进入休眠状态的时间';
```

```sql
CREATE TABLE IF NOT EXISTS public.memory_points_log (

CREATE INDEX IF NOT EXISTS idx_memory_points_log_positive_points ON public.memory_points_log(user_id, points)
```

## 4. 数据库函数

### 4.1 更新记忆点数并检查里程碑

```sql
-- ============================================
-- 更新记忆点数并检查里程碑
-- ============================================
CREATE OR REPLACE FUNCTION public.update_memory_points(
  p_user_id UUID,
  p_points INTEGER,
  p_source_type VARCHAR DEFAULT 'dialogue',
  p_source_detail TEXT DEFAULT NULL
)
RETURNS JSONB
LANGUAGE plpgsql
AS $$
DECLARE
  v_current_points INTEGER;
  v_new_points INTEGER;
  v_milestones JSONB := '[]';
  v_new_abilities JSONB;
BEGIN
  -- 获取当前点数
  SELECT memory_points INTO v_current_points
  FROM public.ai_partners WHERE user_id = p_user_id;
  
  -- 更新点数
  UPDATE public.ai_partners
  SET memory_points = memory_points + p_points,
      updated_at = NOW()
  WHERE user_id = p_user_id;
  
  v_new_points := v_current_points + p_points;
  
  -- 记录日志
  INSERT INTO public.memory_points_log (user_id, points, source_type, source_detail)
  VALUES (p_user_id, p_points, p_source_type, p_source_detail);
  
  -- 检查里程碑并更新能力
  v_new_abilities := '{
    "basic_chat": true,
    "emotion_expression": false,
    "task_system": false,
    "exclusive_memory": false,
    "deep_conversation": false,
    "self_awareness": false
  }';
  
  IF v_new_points >= 5 THEN
    v_new_abilities := jsonb_set(v_new_abilities, '{emotion_expression}', 'true');
    v_milestones := v_milestones || '{"name": "emotion_expression", "points": 5}';
  END IF;
  
  IF v_new_points >= 15 THEN
    v_new_abilities := jsonb_set(v_new_abilities, '{task_system}', 'true');
    v_milestones := v_milestones || '{"name": "task_system", "points": 15}';
  END IF;
  
  IF v_new_points >= 25 THEN
    v_new_abilities := jsonb_set(v_new_abilities, '{exclusive_memory}', 'true');
    v_milestones := v_milestones || '{"name": "exclusive_memory", "points": 25}';
  END IF;
  
  IF v_new_points >= 50 THEN
    v_new_abilities := jsonb_set(v_new_abilities, '{deep_conversation}', 'true');
    v_milestones := v_milestones || '{"name": "deep_conversation", "points": 50}';
  END IF;
  
  IF v_new_points >= 100 THEN
    v_new_abilities := jsonb_set(v_new_abilities, '{self_awareness}', 'true');
    v_milestones := v_milestones || '{"name": "self_awareness", "points": 100}';
  END IF;
  
  -- 更新能力和阶段
  UPDATE public.ai_partners SET
    abilities = v_new_abilities,
    growth_stage = CASE
      WHEN v_new_points >= 500 THEN '觉醒期'
      WHEN v_new_points >= 201 THEN '成熟期'
      WHEN v_new_points >= 51 THEN '成长期'
      ELSE '懵懂期'
    END,
    current_title = CASE
      WHEN v_new_points >= 200 THEN '命运共同体'
      WHEN v_new_points >= 100 THEN '灵魂伴侣'
      WHEN v_new_points >= 50 THEN '默契'
      WHEN v_new_points >= 25 THEN '相知'
      WHEN v_new_points >= 10 THEN '初识'
      ELSE '初识'
    END
  WHERE user_id = p_user_id;
  
  RETURN jsonb_build_object(
    'previous_points', v_current_points,
    'new_points', v_new_points,
    'points_added', p_points,
    'milestones_reached', v_milestones
  );
END;
$$;
```

### 4.2 每周评估函数

```sql
-- ============================================
-- 每周评估函数（简化版）
-- ============================================
CREATE OR REPLACE FUNCTION public.run_weekly_evaluation(
  p_user_id UUID
)
RETURNS JSONB
LANGUAGE plpgsql
AS $$
DECLARE
  v_week_start DATE;
  v_points_grown INTEGER;
  v_result VARCHAR(20);
  v_consecutive_warnings INTEGER;
  v_dormant_since TIMESTAMPTZ;
BEGIN
  -- 计算本周开始日期
  v_week_start := DATE_TRUNC('week', CURRENT_DATE)::DATE;
  
  -- 获取本周新增点数
  SELECT COALESCE(SUM(points), 0) INTO v_points_grown
  FROM public.memory_points_log
  WHERE user_id = p_user_id
    AND created_at >= v_week_start
    AND points > 0;
  
  -- 获取当前休眠状态
  SELECT dormant_since INTO v_dormant_since
  FROM public.ai_partners WHERE user_id = p_user_id;
  
  -- 判定结果
  IF v_points_grown >= 15 THEN
    v_result := 'pass';
    -- 如果处于休眠，唤醒
    IF v_dormant_since IS NOT NULL THEN
      UPDATE public.ai_partners
      SET status = 'normal', dormant_since = NULL
      WHERE user_id = p_user_id;
    END IF;
  ELSE
    v_result := 'warning';
    -- 检查连续警告次数
    SELECT COUNT(*) INTO v_consecutive_warnings
    FROM public.ai_partners ap
    WHERE ap.user_id = p_user_id
      AND ap.status = 'dormant';
    
    -- 连续两周不通过，进入休眠
    IF v_consecutive_warnings >= 1 OR v_dormant_since IS NOT NULL THEN
      UPDATE public.ai_partners
      SET status = 'dormant', dormant_since = NOW()
      WHERE user_id = p_user_id;
    END IF;
  END IF;
  
  RETURN jsonb_build_object(
    'week_start', v_week_start,
    'points_grown', v_points_grown,
    'result', v_result,
    'threshold', 15
  );
END;
$$;
```

### 4.3 每日签到函数

```sql
-- ============================================
-- 每日签到函数
-- ============================================
CREATE OR REPLACE FUNCTION public.process_daily_checkin(
  p_user_id UUID
)
RETURNS JSONB
LANGUAGE plpgsql
AS $$
DECLARE
  v_today DATE := CURRENT_DATE;
  v_yesterday DATE := CURRENT_DATE - INTERVAL '1 day';
  v_last_signin_date DATE;
  v_consecutive_days INTEGER;
  v_base_reward INTEGER := 15;
  v_streak_bonus INTEGER := 0;
  v_total_reward INTEGER;
  v_points_result JSONB;
BEGIN
  -- 检查今天是否已经签到
  SELECT sign_date INTO v_last_signin_date
  FROM public.daily_signin
  WHERE user_id = p_user_id AND sign_date = v_today;
  
  IF v_last_signin_date IS NOT NULL THEN
    -- 今天已经签到
    RETURN jsonb_build_object(
      'success', false,
      'message', '今天已经签到过了',
      'streak_count', 0,
      'total_reward', 0
    );
  END IF;
  
  -- 获取上次签到日期和连续签到天数
  SELECT sign_date, consecutive_days INTO v_last_signin_date, v_consecutive_days
  FROM public.daily_signin
  WHERE user_id = p_user_id
  ORDER BY sign_date DESC
  LIMIT 1;
  
  -- 计算连续签到天数
  IF v_last_signin_date IS NULL THEN
    -- 首次签到
    v_consecutive_days := 1;
  ELSIF v_last_signin_date = v_yesterday THEN
    -- 昨天签到过，连续签到+1
    v_consecutive_days := v_consecutive_days + 1;
  ELSE
    -- 中断了，重新开始
    v_consecutive_days := 1;
  END IF;
  
  -- 计算奖励
  IF v_consecutive_days >= 7 THEN
    v_streak_bonus := 35;  -- 连续7天奖励：15 + 35 = 50
  END IF;
  
  v_total_reward := v_base_reward + v_streak_bonus;
  
  -- 插入签到记录
  INSERT INTO public.daily_signin (
    user_id, sign_date, consecutive_days, points_awarded
  ) VALUES (
    p_user_id, v_today, v_consecutive_days, v_total_reward
  );
  
  -- 更新记忆点数
  SELECT * INTO v_points_result
  FROM public.update_memory_points(p_user_id, v_total_reward, 'signin');
  
  -- 返回结果
  RETURN jsonb_build_object(
    'success', true,
    'streak_count', v_consecutive_days,
    'base_reward', v_base_reward,
    'streak_bonus', v_streak_bonus,
    'total_reward', v_total_reward,
    'memory_points', v_points_result
  );
END;
$$;
```

```sql
CREATE OR REPLACE FUNCTION public.log_audit_changes()
RETURNS TRIGGER
LANGUAGE plpgsql
AS $$
BEGIN
  IF TG_OP = 'INSERT' THEN
    INSERT INTO public.audit_log (table_name, record_id, action, new_values, changed_by)
    VALUES (TG_TABLE_NAME, NEW.id, TG_OP, row_to_json(NEW), current_user);
    RETURN NEW;
  ELSIF TG_OP = 'UPDATE' THEN
    INSERT INTO public.audit_log (table_name, record_id, action, old_values, new_values, changed_by)
    VALUES (TG_TABLE_NAME, NEW.id, TG_OP, row_to_json(OLD), row_to_json(NEW), current_user);
    RETURN NEW;
  ELSIF TG_OP = 'DELETE' THEN
    INSERT INTO public.audit_log (table_name, record_id, action, old_values, changed_by)
    VALUES (TG_TABLE_NAME, OLD.id, TG_OP, row_to_json(OLD), current_user);
    RETURN OLD;
  END IF;
END;
$$;

  AFTER INSERT OR UPDATE OR DELETE ON public.ai_partners
```
