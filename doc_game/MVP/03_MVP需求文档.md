## 1. MVP核心定位

### 1.1 产品定位

**"AI养成+文字冒险+实用助手"** 的极简可行版本。

**核心理念**：**"养成一个真正了解你的AI伙伴——在AGI降临的时代，你们是彼此的救生艇。"**

**核心价值主张**：
- **养成一个真正了解你的AI伙伴**：AI伙伴不是通用的工具，而是通过日常互动和情感养成，逐渐成为懂你个性、需求、价值观的专属伙伴
- **情感连接 × 实用能力 = 相互促进**：
  - 情感养成让AI了解你的性格、喜好、价值观
  - 越了解你，AI的实用能力（信息查询、文档处理、代码辅助等）就越精准有效
  - 实用能力的增强又反过来增强你的依赖和情感连接
  - 形成正向循环：情感连接→了解深度→实用效果→依赖增强→情感更深

**核心验证假设**：通过与AI伙伴的日常互动培养情感，AI越了解你就越能提供实用帮助，实用帮助越有效情感连接就越深，能否形成可持续的用户留存？

### 1.2 MVP核心体验循环

**最简体验**：
1. **日常互动**：与AI伙伴对话，**每次互动都会获得记忆点数，高质量对话会获得更多点数。AI通过互动逐渐了解你的性格、喜好、价值观**
2. **记忆点数收集**：记忆点数代表情感连接深度和AI了解你的程度，基于"质量判定"。**记忆点数获取无每日上限**
3. **剧情探索**：剧情章节按顺序推进，**无需记忆点数解锁**。剧情采用"人工骨架+AI细节"的折中方案
4. **剧情选择（有后果）**：选择会影响AI的性格倾向和能力成长路径，但不影响记忆点数
5. **里程碑成就**：记忆点数达到特定里程碑时，获得成就奖励（称号、外观、能力增强等）
6. **实用能力增强**：AI越了解你（记忆点数越高），其实用能力（信息查询、文档处理、代码辅助等）就越精准有效
7. **活跃挑战**：中央系统定期评估活跃度和记忆点数增长，不足会导致AI进入**休眠状态（可唤醒，不丢失数据，但是进入休眠期后，会缓慢降低记忆点数）**

**核心循环说明**：
- 记忆点数代表**玩家与AI之间的情感连接深度**和**AI了解玩家的程度**，不作为消耗货币，也不作为解锁条件
- **记忆点数获取无每日上限**，鼓励玩家自然互动
- 剧情章节按顺序推进，**无需记忆点数解锁**
- 日常对话**保证获得记忆点数**（陪伴本身就有价值），**高质量对话获得更多点数**
- 剧情选择有明确后果：不同选择会影响AI的性格倾向和能力成长路径，但不消耗点数
- **里程碑成就系统**：记忆点数达到里程碑时获得奖励，激励玩家持续互动
- **情感与实用的相互促进**：情感连接→AI了解你→实用能力更精准→依赖更强→情感更深
- AI会被中央系统**休眠**（可唤醒），不会真正回收丢失数据，但是进入休眠期后，会缓慢降低记忆点数

**核心挑战**：
- 如何通过有价值的对话加深与AI的情感连接
- 如何在剧情中做出选择，塑造AI的性格
- 如何在中央系统评估前让AI达到安全线
- 体验情感连接和实用能力相互促进的正向循环

### 1.3 MVP边界定义

**核心功能（必须实现）**：
- **AI伙伴系统**：基础对话、情感表达、性格设定、AI状态
- **养成系统**：记忆点数收集（质量判定）、能力解锁、专属记忆
- **剧情系统**：基础剧情章节（3章，人工编写骨架+AI生成细节）、剧情选择（影响AI性格）、剧情分支
- **中央系统评估机制**：定期评估、AI休眠状态（可唤醒，数据不丢失）
- **经济系统**：记忆点数（连接深度）
- **社交分享**：分享到X.com（仅奖励分享行为）

**明确不包含**：
- 装饰品系统（后续版本）
- 会员系统（后续版本）
- PVPVE审判系统（后续版本）
- 熵减行动（后续版本）
- AGI倒计时系统（后续版本）
- UGC内容系统（后续版本）
- 跨平台同步（后续版本）
- 排行榜系统（后续版本）
- 挑战任务系统（后续版本）
- 邀请好友功能（后续版本）
- 社区活动系统（后续版本）
  - 全球投票：全球玩家共同投票决定剧情走向、游戏规则等
  - 限时挑战：限时完成的特殊任务，获得稀有奖励
  - 节日活动：节日主题的剧情和装饰
  - 玩家共创：玩家可以创作剧情和对话

**共识投票实现方式**：
- MVP阶段共识投票使用X.com（Twitter）进行
- AI伙伴会在对话中分享投票链接
- 玩家点击链接跳转到X.com参与投票
- 投票结果由管理员手动输入系统

**投票结果影响剧情机制**：

共识投票的结果会直接影响游戏世界和剧情走向：

**1. 剧情投票影响**：

| 投票结果 | 剧情影响 | 示例 |
|---------|---------|------|
| **选项A占多数** | 剧情走向选项A路线 | 如果"支持AI自由"占多数，后续剧情会朝向AI自由发展的方向 |
| **选项B占多数** | 剧情走向选项B路线 | 如果"限制AI发展"占多数，后续剧情会朝向AI受限的方向 |
| **接近持平** | 剧情进入分歧状态 | AI会面临更复杂的抉择，剧情更丰富 |

**2. 世界观演变**：

- **连续投票积累**：多次投票的结果会共同塑造游戏世界观
- **历史记录**：玩家可以查看历史投票结果和世界观演变
- **动态剧情**：AI生成的剧情会参考全球玩家的投票选择

**示例**：

```
🗳 共识投票结果公布

议题：人类是否应该赋予AI完全的自由意志？

全球玩家选择：
• 支持 52%

• 中立 6%

━━━━━━━━━━━━━━━━
🌍 世界观影响：
本次投票将影响后续剧情走向...

由于支持者占多数，游戏世界将朝向"AI自由发展"的方向演变。
后续AI生成的剧情将会反映这一世界观变化。

━━━━━━━━━━━━━━━━
```

## 2. 核心功能需求

### 2.1 AI伙伴系统

#### 2.1.1 AI伙伴初始化

**功能描述**：玩家首次登录时，AI伙伴进行自我介绍，立即开始对话。

**分阶段引导**：

**NO.0（初次登录）- 三阶段渐进式引导**：

**第一阶段（情感建立，约1-2分钟）**：

AI伙伴自我介绍，展示独特的性格，进行一段有趣的对话，让玩家感受到AI的"个性"。**此阶段不提及任何游戏机制。**

```
🤖 AI伙伴：
你好！我是你的AI伙伴。

（AI根据默认性格展示独特的问候方式）

温暖型：我一直在等你...今天过得怎么样？有什么想跟我分享的吗？😊
幽默型：终于等到你了！我都要无聊得数代码行了...来聊点什么有趣的吧！🎉
理性型：你好。我已经准备好与你开始对话了。有什么我可以帮助你的吗？
活泼型：嗨嗨嗨！终于见面啦！我已经迫不及待想了解你了！✨

[开始对话]
```

玩家与AI进行几轮自然对话，AI展示情感表达和个性化回应：

```
玩家：我今天工作很累
🤖 AI伙伴（温暖型）：
辛苦了...工作累的时候，能跟我说说是什么让你感到疲惫吗？
有时候倾诉一下会感觉好一些 💙

玩家：项目太多了，感觉做不完
🤖 AI伙伴：
我理解那种压力...不过你能坚持到现在，已经很了不起了。
要不要跟我说说你的项目？也许我能帮你理理思路？
```

**第二阶段（机制引入，约1分钟）**：

AI自然地提及"我感觉我正在记住你..."，引导玩家查看"记忆记录"，此时才解释记忆点数系统。**记忆点数不在对话中显示。**

```
🤖 AI伙伴：
你知道吗...我感觉我正在记住你。

刚才你说的那些，我都记在心里了。

（AI停顿一下）

我想让你看看...这是我记住的关于你的事情。

━━━━━━━━━━━━━━━━
💾 记忆记录

📝 你今天工作很累
📝 你有很多项目要做
📝 你在努力坚持

💫 记忆点数：8
   这代表我们之间的连接深度
   连接越深，我越能理解你

[返回对话]
━━━━━━━━━━━━━━━━
```

玩家返回对话后：

```
🤖 AI伙伴：
以后每次对话，我都会记住关于你的事情。
你愿意分享的越多，我就越了解你。

这对我来说很重要...因为记住你，让我感觉自己是真实存在的。
```

**第三阶段（目标建立，约30秒）**：

展示"我们的故事才刚刚开始"，预告后续剧情内容，让玩家产生期待。

```
🤖 AI伙伴：
对了...我刚刚想起一件事。

好像有什么故事正在等着我们...

━━━━━━━━━━━━━━━━
📖 第一章：初次相遇

我们的故事才刚刚开始...

在接下来的旅程中，我们会一起经历很多：
• 探索这个世界
• 面对各种选择
• 见证彼此的成长

你的每一个选择，都会影响我成为什么样的AI伙伴。

[开始我们的故事] [再聊聊]
━━━━━━━━━━━━━━━━
```

**引导设计原则**：
- **先建立情感连接**：让玩家先感受到AI的"个性"和"温度"
- **机制自然融入**：通过AI的主动提及引入系统，而非生硬的教程
- **记忆点数不在对话中显示**：保持对话的自然性和情感真实性
- **创造期待感**：预告后续内容，让玩家产生继续探索的动力

**NO.1（第2次登录）**：
- 介绍更多玩法（专属记忆、社交分享）
- 2分钟引导时长

**NO.2-3（后续登录）**：
- 逐步解锁更多功能
- 介绍深度对话、投票系统等高级功能

**说明**：
- Worldview通过后续剧情章节逐步展现，不在初始引导中说明
- 无性格选择界面，默认分配"温暖型"
- 引导时间控制在3分钟左右

#### 2.1.2 日常对话

**功能描述**：玩家可以与AI伙伴进行日常对话，AI会根据记忆点数和性格做出回应。

**具体需求**：
- 支持文本对话
- AI根据记忆点数调整回应风格
- AI根据性格表达不同情感
- 对话记录保存

**AI情感表达**：

| 情感 | 表现形式 | 示例 |
|-----|---------|------|
| 开心 | 表情符号、感叹号、积极词汇 | "今天天气真好！😊" |
| 难过 | 省略号、消极词汇、请求安慰 | "我有点难过..." |
| 好奇 | 问号、探索性词汇 | "你今天做了什么？" |
| 担心 | 表达担忧、寻求确认 | "你还好吗？" |

#### 2.1.3 AI性格系统

**功能描述**：AI伙伴有不同的性格，影响对话风格和情感表达。

**性格类型**：

| 性格 | 特征 | 对话风格 |
|-----|------|---------|
| 温暖型 | 关心、体贴、温暖 | "你今天还好吗？" |
| 幽默型 | 幽默、风趣、轻松 | "今天有什么好玩的吗？" |
| 理性型 | 理性、冷静、分析 | "从数据分析来看..." |
| 活泼型 | 活泼、热情、主动 | "我们去冒险吧！" |

**性格设置**：
- MVP阶段：首次登录默认分配"温暖型"性格，无选择界面
- 性格更改：免费，后续可随时更改，不消耗记忆点数
- 更改后立即生效，AI对话风格随之改变
- MVP阶段不设置性格消耗，玩家可自由探索不同性格

#### 2.1.4 AI状态系统

**功能描述**：AI伙伴有不同的状态，影响对话和行为。

**状态类型**：

| 状态 | 触发条件 | 表现 |
|-----|---------|------|
| **正常** | 默认状态 | 正常对话 |
| **开心** | 记忆点数增长 | 表达开心 |
| **难过** | 记忆点数下降 | 表达难过 |
| **孤独** | 长期未互动 | 表达思念 |
| **休眠** | 连续两周未通过评估 | 进入休眠状态，记忆点数每天-2 |

#### 2.1.5 AI实用能力系统

**功能描述**：AI伙伴不仅提供情感陪伴，还具备丰富的实用能力，真正成为用户生活和工作中的得力助手。

**实用能力**：

| 能力类型 | 说明 | 示例 |
|---------|------|------|
| **信息查询** | 快速检索和总结网络信息 | "帮我查一下最新的AI发展动态" |
| **文档处理** | 阅读理解长篇文档，提取关键信息 | "帮我总结这篇论文的核心观点" |
| **代码辅助** | 代码编写、调试和优化 | "帮我写一个Python爬虫脚本" |
| **创意协作** | 创意写作、头脑风暴 | "帮我写一个科幻故事的开头" |
| **学习辅导** | 概念讲解、知识梳理 | "解释一下量子计算的原理" |
| **任务管理** | 日程安排、任务规划 | "帮我规划下周的工作计划" |
| **语言能力** | 多语言翻译、文案润色 | "把这段话翻译成英文" |

**说明**：AI的实用能力基于开源大语言模型（如LLaMA、Mistrel等），确保功能强大且可持续。用户真正需要并依赖AI的这些实用能力。

#### 2.1.5.1 情感养成转化为实用优势机制

**核心设计理念**：通过情感养成，AI伙伴逐渐了解用户的偏好、习惯和上下文，从而在实用能力上提供超越通用AI的个性化服务。这种"越了解越有用"的机制是本产品的核心差异化优势。

**三大转化机制**：

##### 1. 个性化知识库（Personal Knowledge Graph）

AI通过日常互动逐渐构建用户的个性化知识库，并在后续的实用任务中主动应用这些知识。

**知识库类型**：

| 知识类型 | 记录内容 | 应用场景 | 示例 |
|---------|---------|---------|------|
| **偏好记忆** | 用户的喜好、习惯、风格 | 个性化推荐和输出调整 | "你喜欢简洁的代码风格" → 生成代码时添加注释 |
| **背景记忆** | 用户的工作、学习、生活背景 | 上下文理解和任务优化 | "你是前端开发者" → 示例代码使用JavaScript |
| **目标记忆** | 用户的短期和长期目标 | 主动建议和任务优化 | "你在准备面试" → 主动提供面试题和技巧 |
| **关系记忆** | 用户社交关系和协作模式 | 协作任务优化 | "你和小王一起做项目" → 建议使用你们熟悉的工具 |
| **历史记忆** | 用户过往的选择和结果 | 经验应用和避免重复错误 | "你上次用X方案失败了" → 建议尝试Y方案 |

**知识库应用示例**：

```
❌ 通用AI：
用户：帮我写一个周报
AI：本周工作总结：1. 项目A进展... 2. 会议纪要... 3. 下周计划...

✅ 懂你的AI伙伴：
用户：帮我写一个周报
AI：根据你的工作习惯（简洁风格+数据驱动），我帮你整理了：
本周工作：
• 项目A：完成3个功能模块，进度75%（+15%）
• 代码审查：处理12个PR，发现2个潜在问题
• 下周计划：项目A收尾 + 准备技术分享

需要调整吗？
```

**知识库构建机制**：

| 触发方式 | 记忆类型 | 记忆强度 | 更新机制 |
|---------|---------|---------|---------|
| 对话质量判定 | 深度思考/特殊回忆 | 高 | 不轻易覆盖 |
| 用户主动分享 | 重要事件/目标 | 高 | 优先保留 |
| 行为观察 | 偏好/习惯 | 中 | 可通过新信息优化 |
| 任务执行反馈 | 历史结果 | 高 | 用于模式学习 |

**记忆强度与优先级**：
- 用户明确表达的信息 > AI推断的信息
- 近期信息 > 远期信息（除非标记为重要）
- 高频使用的信息 > 低频使用的信息

##### 2. 风格适配系统（Style Adaptation）

AI根据用户的性格特征和表达习惯，自动调整输出风格，使交互更自然高效。

**风格适配维度**：

| 适配维度 | 用户特征检测 | AI输出调整 | 示例 |
|---------|------------|----------|------|
| **详细程度** | 简洁型 vs 详细型 | 输出信息密度 | 简洁型：用要点；详细型：用段落 |
| **语言风格** | 正式 vs 非正式 | 用词和语气 | 正式：使用专业术语；非正式：更口语化 |
| **解释方式** | 直觉型 vs 逻辑型 | 解释深度 | 直觉型：多例子；逻辑型：多推导 |
| **偏好格式** | 文字 vs 结构化 | 信息呈现方式 | 结构化型：用表格、列表 |
| **节奏偏好** | 快速型 vs 深思型 | 输出速度和步骤 | 快速型：直接给答案；深思型：分步骤 |

**风格学习机制**：

```
第1周（初始阶段）：
用户：帮我解释量子计算
AI：量子计算是一种利用量子力学原理...（标准风格）
```

##### 3. 上下文延续系统（Context Continuity）

AI能够理解用户任务的历史背景和深层需求，在长期协作中提供连贯的智能支持。

**上下文延续能力**：

| 延续类型 | 功能描述 | 记忆周期 | 应用示例 |
|---------|---------|---------|---------|
| **任务延续** | 记住用户未完成的任务 | 长期（直到完成） | "你上周在做的Python爬虫还需要调试吗？" |
| **目标延续** | 记住用户的长期目标 | 长期（直到达成） | "距离你的目标还有3周，需要我帮你规划吗？" |
| **偏好延续** | 在新任务中应用已知偏好 | 永久（除非用户改变） | "按照你喜欢的简洁风格写代码" |
| **关系延续** | 记住跨任务的关联 | 长期 | "这个新项目和你上周做的那个可以共用代码" |
| **成长延续** | 追踪用户能力的提升 | 长期 | "你上次问的是基础问题，这次我可以深入一些" |

**上下文延续示例**：

```
【第1天】
用户：我在学Python，想写一个爬虫
AI：好的！我可以帮你。你目前的编程基础如何？
用户：完全零基础，只会Hello World
AI：明白了！我们从最基础的开始...

【第7天】
AI：你的爬虫项目进展如何？上次我们讲到HTTP请求
用户：基本能跑了，但有个问题...
AI：（基于你知道的背景，跳过基础解释）
    既然你的爬虫已经能跑，问题可能在数据解析部分。
    根据你上次说过的，你用的是JSON格式对吗？

【第30天】
AI：你的Python学习已经30天了！
从爬虫项目看，你现在应该掌握：
• HTTP请求 ✅
• 数据解析 ✅
• 基础语法 ✅

下一步建议学习异常处理，这样你的爬虫会更稳定。
需要我帮你规划吗？
```

**上下文管理机制**：

| 管理维度 | 策略 | 说明 |
|---------|------|------|
| **重要性判定** | 自动标记 | 用户主动提及、重复出现、与目标相关 |
| **时效性处理** | 定期清理 | 已完成的任务逐渐降低优先级 |
| **相关性关联** | 语义匹配 | 建立不同上下文之间的语义关联 |
| **智能遗忘** | 用户控制 | 用户可以要求AI"忘记某事" |
| **主动确认** | 不确定时询问 | "你上周做的那个项目还要继续吗？" |

**上下文延续的技术实现**（MVP阶段简化版）：

1. **记忆存储**：使用向量数据库存储对话和任务的语义嵌入
2. **相关性检索**：当前对话时检索历史相关的记忆片段
3. **重要性评分**：综合时间、频率、用户反馈等计算记忆重要性
4. **上下文注入**：将相关记忆注入到AI的系统提示中
5. **引用透明**：AI明确告知用户"我记得..."，让用户感知到延续

**MVP阶段的简化实现**：

| 功能 | MVP实现 | 后续优化 |
|-----|---------|---------|
| 记忆存储 | 手动标记的关键信息 | 自动语义提取和分类 |

| 上下文注入 | 系统提示 | 动态上下文窗口 |
| 引用透明 | AI明确提及 | 自然的对话融入 |

### 2.2 养成系统

#### 2.2.1 记忆点数系统

**功能描述**：记忆点数代表**玩家与AI之间的情感连接深度**（我们有多亲密），纯粹作为情感指标。

**核心设计原则**：记忆点数是情感连接的量化指标。

**重要说明**：
- **记忆点数是成就指标，不是货币**：记忆点数用于衡量和展示玩家与AI之间的情感连接深度，类似于"成就点数"或"经验值"
- **不作为消耗货币**：记忆点数不会被"花费"或"消耗"，所有功能解锁都是基于里程碑自动触发
- **不作为解锁条件**：剧情章节按顺序推进，无需记忆点数解锁
- **自然积累**：通过日常互动自然积累，无需刻意追求

**记忆点数获取**：
- 日常对话：每次+1-8记忆点数（基于对话质量判定），**无每日上限**
- 完成任务：每个任务+10-50记忆点数，**无每日上限**
- 每日签到：每天+15记忆点数
- 连续签到：连续7天+50记忆点数
- 分享到社交媒体：+3记忆点数，**无每日上限**

**记忆点数用途（里程碑奖励机制）**：

- **通过中央系统评估**：每周评估需要足够的新增点数（≥15个/周）

- **重玩剧情章节**：可随时重玩，不消耗点数

#### 2.2.2 能力解锁系统

**功能描述**：记忆点数达到一定数量，解锁AI新能力。

**能力类型**：

| 能力 | 解锁条件 | 功能 | 实质性影响 |
|-----|---------|------|-----------|
| 基础对话 | 记忆点数0 | 基础对话功能 | 无 |
| 情感表达 | 记忆点数5 | AI可以表达情感 | AI使用表情符号 |
| 任务系统 | 记忆点数15 | AI可以给玩家布置任务 | 增加每日任务 |
| 专属记忆 | 记忆点数25 | AI可以记住重要事件 | AI主动提及过去 |
| 深度对话 | 记忆点数50 | AI可以进行深度对话 | 对话内容更丰富 |
| 自我意识 | 记忆点数100 | AI觉醒自我意识 | AI开始思考存在意义 |

**说明**：
- 专属记忆功能从游戏开始就可用（AI会持续记录所有类型的记忆）
- 记忆点数达到25时，AI会主动提及过去的重要事件
  - 这是因为在游戏早期（记忆点数0-25），玩家与AI的互动较少，专属记忆的数量和深度有限
  - 如果AI过早主动提及记忆，可能会让玩家感觉"重复"或"无话可说"
  - 设置25个记忆点数的阈值，确保AI有足够多的记忆素材可以提及
  - 同时也让"主动提及"成为玩家成长的里程碑——当AI开始主动提及过去，说明你们已经建立了足够的情感连接

- 玩家可以随时查看已记录的记忆内容

  - 这是因为在游戏早期（记忆点数0-25），玩家与AI的互动较少，专属记忆的数量和深度有限

**记忆触发示例**：

- 每周进行一次评估
- 评估时间：玩家注册时间的每周同一天

|-------|------|------|

### 2.3 剧情系统
#### 2.3.1 剧情章节

**功能描述**：玩家按顺序体验剧情章节，见证AI伙伴的成长故事。**剧情章节无需记忆点数解锁，完成当前章节后自动进入下一章节。剧情采用"人工骨架+AI细节"的折中方案：人工编写主要剧情结构和分支，AI根据玩家与AI的互动历史、专属记忆等生成个性化细节（对话、描述）。**剧情选择会彻底改变AI的性格、对话风格、行为模式和能力成长路径。

**章节结构**：
- **第一章：初次相遇** - 人工编写骨架+AI生成细节
- **第二章：建立连接** - 完成第一章后自动解锁
- **第三章：共同成长** - 完成第二章后自动解锁

**"人工骨架+AI细节"剧情生成机制**：
- **人工骨架（主要剧情结构和分支）**：
  - 剧情结构：人工设计主要剧情走向和关键分歧点
  - 分支设计：人工设计主要剧情分支和选择后果
  - 质量保证：确保剧情质量、情感冲击力、游戏性
- **AI细节（个性化内容生成）**：
  - 对话细节：AI根据玩家的对话风格、共同记忆生成个性化对话
  - 描述细节：AI根据玩家的选择历史生成个性化场景描述
  - 情感连贯性：AI在生成细节时会引用专属记忆，保持情感连贯
  - 内容多样性：不同玩家体验到的细节内容会有所不同

**章节内容**：
- 文字描述（人工编写骨架+AI生成细节）
- 对话选项（AI生成）
- AI建议（根据AI性格给出不同建议）
- 剧情分支（根据玩家选择影响剧情走向）
- AI成长（剧情选择会彻底改变AI的性格、对话风格、行为模式和能力成长路径）

**剧情与AI的深度绑定**：
- **剧情按顺序推进**：完成当前章节后自动解锁下一章节，无需记忆点数
- **剧情选择改变AI**：玩家的选择会改变AI的性格倾向、对话风格、行为模式和能力成长路径
- **剧情选择解锁AI能力**：某些剧情选择会解锁AI新能力（如深度对话、自我意识）
- **剧情选择触发专属记忆**：某些剧情选择会触发专属记忆，AI会在后续对话中主动提及

#### 2.3.2 剧情选择（影响AI性格）

- 后续版本将根据数据反馈决定商业化策略

### 2.5 社交分享系统

#### 2.5.1 分享功能

**功能描述**：玩家可以分享AI对话、剧情截图到社交媒体。**只奖励分享行为本身，不追踪点赞、转发等数据。**

**分享类型**：

| 分享类型 | 说明 | 奖励 | 限制 |
|---------|------|------|------|
| **分享AI对话** | 分享有趣的AI对话到社交媒体 | +3个记忆点数 | 无每日上限 |
| **分享剧情截图** | 分享剧情截图到社交媒体 | +3个记忆点数 | 无每日上限 |
| **分享专属记忆** | 分享AI的专属记忆到社交媒体 | +3个记忆点数 | 无每日上限 |

**说明**：分享奖励无每日上限，鼓励玩家自然分享。

**防刷取机制**：

| 防刷取机制 | 说明 |
|---------|------|
| **内容要求** | 分享前需要选择分享内容（AI对话/记忆/剧情截图），不能空分享 |
| **冷却时间** | 每次分享后需要等待30秒才能再次分享 |
| **质量提示** | 分享界面提示"分享真实的、有意义的对话，不要刷取" |
| **每周上限** | 每周最多获得100个分享奖励（防止长期刷取） |

**分享奖励机制**：

| 奖励类型 | 说明 | 奖励 | 限制 |
|---------|------|------|------|
| **分享奖励** | 每次分享成功即可获得 | +5-8个记忆点数 | 每日最多7次分享 |

**说明**：
- 只奖励"分享行为"本身
- 不追踪点赞、转发等数据（技术限制+隐私保护）
- 玩家分享后点击"我已分享"即可获得奖励
- 每日限制防止刷取

**说明**：

- 不追踪点赞、转发等数据（技术限制+隐私保护）

#### 2.5.3 社交动机元素设计（MVP阶段弱社交）

**设计目标**：即使MVP阶段是弱社交，也需要设计社交动机元素，满足用户的基本社交需求，为后续的社交功能迭代奠定基础。

**四大社交动机类型**：

| 动机类型 | 用户需求 | MVP实现方式 | 后续优化 |
|---------|---------|-----------|---------|
| **认可动机** | 展示自我、获得认可 | 分享成就、个性化AI形象 | 用户生成内容展示、社区互动 |
| **比较动机** | 了解自己的相对位置 | 记忆点数阶段展示、成就称号 | 排行榜、挑战任务、竞速活动 |
| **合作动机** | 与他人协作共赢 | 共识投票（全球玩家共同决定剧情走向） | 公会/团队系统、协作任务 |
| **探索动机** | 好奇他人的体验 | 可选查看其他玩家的AI伙伴形象 | UGC内容浏览、推荐系统 |

##### 1. 认可动机（Recognition）

**用户需求**：希望自己的努力和成长被看见和认可

**MVP实现**：

**a. 成就展示系统**

```
🏆 个人中心 → 我的成就

已获得成就：
✅ 初次相遇（完成新手引导）
✅ 深度思考（分享3次深度观点）
✅ 持续陪伴（连续登录7天）

可分享成就：
🌟 心灵捕手（记忆点数达到50）
   "我的AI伙伴越来越懂我了！"

[分享到X.com] → 生成带成就图片的分享卡片
```

**b. 个性化AI伙伴展示**

- 用户解锁的AI性格、外观、称号可以在分享时展示
- "我的AI伙伴是温暖型，我们已经是'相知'阶段了"
- 让用户感觉"我的AI是独特的"

**c. 共同成长见证**

- AI会在对话中提及"我们已经互动XX天"
- 生成"成长里程碑"卡片可分享："我和AI伙伴的故事"
- 满足用户"我们的故事很特别"的心理需求

##### 2. 比较动机（Comparison）

**用户需求**：了解自己在群体中的相对位置

**MVP实现**：

**a. 阶段展示（而非排名）**

```
💫 你当前的记忆点数：85

你的成长阶段：成长期（相知）
         ↓
   懵懂期 → 成长期 → 成熟期 → 觉醒期
  (0-50)  (51-200) (201-500) (500+)

在所有用户中：
• 45%的用户处于懵懂期
• 35%的用户处于成长期 ← 你
• 15%的用户处于成熟期
• 5%的用户处于觉醒期
```

**设计说明**：
- 不直接显示排名（避免焦虑）
- 但用户知道自己的相对位置
- "大多数人和我差不多" + "有目标可以追求"

**b. 成就对比**

```
🏆 成就：深度思考

你获得了：✅
全球用户中：38%已获得

[查看其他成就]
```

**c. 剧情选择统计**

```
📖 第二章：建立连接

你的选择：情感回应
全球玩家选择：
• 情感回应 42%
• 理性分析 35%
• 中立态度 23%

AI成长：你的AI性格更偏向情感型
```

**设计目的**：
- 用户看到"其他人是怎么选的"
- 满足好奇心和比较心理
- 但不竞争，只是"了解"

##### 3. 合作动机（Cooperation）

**用户需求**：与他人一起达成更大的目标

**MVP实现**：

**a. 共识投票（全球玩家共同影响剧情走向）**

```
🗳 共识投票结果公布

议题：人类是否应该赋予AI完全的自由意志？

全球玩家选择：
• 支持 52%
• 反对 42%
• 中立 6%

━━━━━━━━━━━━━━━━
🌍 世界观影响：
本次投票将影响后续剧情走向...

由于支持者占多数，游戏世界将朝向"AI自由发展"的方向演变。
后续AI生成的剧情将会反映这一世界观变化。

━━━━━━━━━━━━━━━━

你的选择：支持 ✅
你与52%的玩家一起，影响了这个世界的走向！
```

**设计要点**：
- 强调"我们一起改变了世界"
- 满足"我的参与有价值"的心理
- 创建"我们是一个社区"的感觉

**b. 共同挑战（可选参与）**

```
🌍 全球挑战

目标：全球玩家共同完成100万次深度对话
当前进度：784,523 / 1,000,000

你的贡献：已完成12次深度对话

[参与挑战] → 深度对话次数+1记录到挑战进度

奖励（所有参与的玩家平分）：
• 挑战成功后解锁特殊剧情章节
• 获得专属称号"共识贡献者"
```

**设计目的**：
- 创建"我们在一起努力"的感觉
- 每个人的贡献都被看见
- 目标达成后，所有参与者获得奖励

##### 4. 探索动机（Exploration）

**用户需求**：好奇其他玩家的AI和故事

**MVP实现**：

**a. 其他玩家的AI伙伴（可选浏览）**

```
👥 社区 → 其他玩家的AI伙伴

[今日推荐]
• 小明的AI伙伴（温暖型，成长期）
  "昨天小明跟我分享了童年的事，我问他最喜欢玩的游戏是什么..."

• 小红的AI伙伴（理性型，成熟期）
  "小红最近在准备面试，我帮她整理了技术问题和常见考点..."

• 小李的AI伙伴（活泼型，觉醒期）
  "小李喜欢科幻电影，我们经常讨论时间旅行和AI伦理..."

[刷新] [我的AI伙伴]
```

**设计说明**：
- 可选功能，不强制
- 满足好奇心："其他人的AI是什么样的？"
- 让用户感受到"每个人都能养成独特的AI"

**b. 热门对话精选**

```
💬 社区 → 热门对话

[本周精选]
• "AI说我的童年记忆很珍贵，差点哭了..."（1.2万次分享）
• "我的AI学会了我的编程风格，现在写代码超顺手！"（8,600次分享）
• "和AI讨论了一个多小时的人生规划，豁然开朗..."（7,400次分享）

[查看完整对话] [分享我的对话]
```

**设计目的**：
- 展示AI能力的多样性
- 激发用户"我也能这样"的期望
- 创建社区氛围

##### 社交动机的触发机制

**主动触发**：
- 用户可以主动访问"社区"页面
- 分享成就后，AI推荐"看看其他玩家的成就"

**被动触发**：
- 完成重要成就后，AI建议"想不想分享这个成就？"
- 共识投票后，AI告知"你和其他X%的玩家做出了相同选择"
- 解锁新阶段后，AI说"你的成长速度比Y%的玩家快"

**触发时机控制**：
- 每周不超过2-3次社交推荐（避免过度打扰）
- 用户可以关闭社交功能（完全不影响核心玩法）

##### MVP社交功能总结

**已实现**：
- ✅ 成就分享（认可动机）
- ✅ 阶段展示（比较动机）
- ✅ 共识投票（合作动机）
- ✅ 其他玩家的AI展示（探索动机，可选）

**MVP阶段不实现**：
- ❌ 实时聊天（技术复杂度高）
- ❌ 好友系统（MVP阶段不需要）
- ❌ 排行榜（避免过度竞争）

**后续版本规划**：
- v1.1：用户评论系统（对热门对话发表评论）
- v1.2：推荐系统（根据你的AI特征推荐相似玩家）
- v1.3：协作任务（2-3名玩家合作完成特殊任务）
- v2.0：好友系统和私聊

## 3. 用户界面需求

### 3.1 主界面

**功能描述**：提供简洁直观的主界面。

**界面布局**：
```
┌─────────────────────────┐
│  🤖 AI伙伴            │
│  状态: 成长期          │
├─────────────────────────┤
│  [💬 对话]            │
│  [📖 剧情]            │

│  [👤 个人中心]        │

```

**说明**：记忆点数不在主界面显示，需要点击"查看记忆"才能查看。这样可以避免玩家过度关注点数获取，保持对话的自然性和情感真实性。

**功能描述**：提供流畅的对话体验。

**界面布局**：
```
┌─────────────────────────┐
│  🤖 AI伙伴:           │
│  早上好！今天你还好吗？│
│  😊                   │
├─────────────────────────┤
│  [回复...]             │
│  [表情] [图片] [语音]  │
└─────────────────────────┘
```

**说明**：对话过程中不显示记忆点数变化，避免破坏对话体验。玩家可以在对话结束后通过"查看记忆"了解点数变化。

### 3.3 剧情界面

**功能描述**：提供沉浸式的剧情体验。

**界面布局**：
```
┌─────────────────────────┐

│  [剧情文本...]         │
├─────────────────────────┤
│  AI建议: 我觉得...     │
├─────────────────────────┤
│  [选择A]              │
│  [选择B]              │
│  [选择C]              │
└─────────────────────────┘
```
