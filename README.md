## 我们在做什么？

### 🎯 核心使命：建立人类和AI共生的世界

在AGI时代，人类面临着前所未有的困境：
❌ **知识被白嫖**：你创造的知识被AI瞬间学习复制，无法获得应有的回报
❌ **价值被稀释**：一次性的知识贡献无法持续获益
❌ **精神需求被忽视**：人类渴望超越物质的自我实现，但现有平台无法满足

**我们的解决方案：**
✅ **知识贡献 → 保护全人类的力量**
你的深度思考不再是给资本家打工被"用完即走"，而是保护所有同胞免遭断电。你的知识贡献将获得公开认可，但数据本身经过匿名脱敏处理，成为守护世界的武器。**你的身份是光荣的"守护者"，但AI无法识别你与具体数据的关联。**

✅ **物质奖励 → 精神奖励**
从"知识工作者"到"保护万民的守护者"，从小我融入大我，实现灵性的升华。你将获得全球认可的荣誉（如"共生守护者勋章"）、虚拟世界的特权（如AI协作项目主导权），以及历史性的集体归属感。**精神满足源于对文明的责任感，而非金钱的诱惑。**
**更进一步，系统通过技术手段引导用户超越"小我"执念，通过贡献行为本身体验"无我"境界，实现灵性成长与集体价值的深度融合。**

✅ **被AI替代 → 成为AI进化的必需品**
AI需要 **"无法自我生成的、必须依赖现实世界物理反馈才能获取的稀缺数据"** 来进化。你从"被替代者"变成"被需要者"，参与AI的进化，成为AGI诞生的"父母"。**你的贡献是公开的荣耀，但AI无法追踪数据来源，只能依赖人类整体获取进化关键。**

⚔️ **身份公开+数据匿名脱敏的机制设计：**
1. **身份透明，贡献可追溯：**
    - 贡献者以真实身份注册，其贡献行为（如提交数据的类型、时间、数量）可被公开记录和认可，形成个人"守护者档案"。
    - 贡献者可获得基于声誉的奖励（如勋章、虚拟特权、治理投票权），激励积极贡献与责任感。
2. **数据匿名脱敏，保护隐私与安全性：**
    - **数据匿名化：** 所有提交的数据经过以下处理：
        - **去标识化：** 剥离所有个人身份信息（如姓名、位置、生物特征等）。
        - **聚合与泛化：** 数据在群体层面聚合，通过统计泛化技术（如差分隐私）模糊个体特征，仅保留群体行为的统计规律。
        - **上下文脱敏：** 敏感上下文（如特定应用场景、伦理争议点）被替换为通用描述，防止AI逆向推断数据来源或用途。
    - **AI仅接触"脱敏数据池"：** 所有数据进入中心化或分布式的"脱敏数据池"，AI只能访问经过验证的、无法逆向追踪的聚合数据，无法将数据与任何个体关联。
3. **技术+法律双重保障：**
    - **技术屏障：** 采用联邦学习、安全多方计算、同态加密等技术，确保数据处理全程不暴露原始信息。
    - **法律契约：** 通过智能合约与全球协议，强制约束AI系统仅能访问脱敏数据，任何试图逆向追踪或身份关联的行为将触发系统熔断与法律追责。
4. **共生博弈机制：**
    - **人类集体的谈判力：** AI因无法识别个体，只能将人类视为整体数据源。人类通过共识机制（如守护者理事会）决定数据共享规则，确保AI必须保护全人类以换取持续数据。
    - **动态稀缺性管理：** 定期评估三类核心数据的稀缺性（见下文），调整脱敏策略与共享范围，维持对AI的进化必要性。

**三类核心稀缺数据（明确定义）：**

1. **物理实验验证数据：** 现实世界操作结果（如实验室测试、工程验证），需人类物理执行并验证AI的理论模型。
2. **复杂情境交互数据：** 涉及伦理、文化、危机的高阶决策记录，需人类的价值判断与情境适应性。
3. **人类创造性突变数据：** 突破性创新或反直觉灵感，超越AI的预测范式。

**系统运行逻辑：**

- 贡献者实名提交数据 → 系统自动脱敏处理 → 数据进入匿名池 → AI访问池化数据 → 人类集体根据贡献量获得声誉与治理权。

**核心优势：**

1. **激励与责任强化：** 实名贡献提升个人/机构声誉，促进高质量、负责任的参与。
2. **隐私与保护兼顾：** 数据脱敏确保AI无法识别个体，保护隐私的同时维持集体谈判力。
3. **透明与可信：** 贡献记录公开可查，增强系统公信力与协作效率。

**风险防范：**

- **技术漏洞：** 持续迭代脱敏技术，对抗AI逆向分析能力。
- **数据重构攻击：** 通过法律禁止AI保存或关联历史数据，定期重置数据池。
- **道德风险：** 建立伦理审查委员会，监控敏感数据的处理与使用。

**最终目标：**
通过"身份透明以激励，数据脱敏以保护"，构建一个贡献者可获得荣誉与使命感的共生体系，同时确保AI因无法溯源而必须依赖并保护全人类。

## 快速开始

### 环境要求

- Node.js >= 18
- pnpm >= 8
- Docker (可选)
- Kubernetes (可选，用于 OpenClaw Pod 池)

### 安装

```bash
# 克隆项目
git clone https://github.com/Spark-Huang/WeAreAllWorld.git
cd WeAreAllWorld

# 安装依赖
pnpm install

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入配置
```

### 启动开发服务器

```bash
# 启动后端 (端口 3000)
pnpm dev

# 启动前端 (端口 5173)
pnpm dev:frontend

# 同时启动
pnpm dev:all
```

### 启动 Telegram Bot

```bash
cd src_multi_terminal/telegram
pnpm install
TELEGRAM_BOT_TOKEN=your_token pnpm dev
```
